# 表单验证引擎和预览交互功能自测文档

## 文档概述

本文档记录了表单验证引擎和预览模式表单交互功能的测试步骤和预期结果。

---

## 第一部分：验证引擎核心逻辑测试

### 测试 1：required 规则 - 空值校验

**测试目的**：验证 required 规则能够正确检测空值并返回错误。

**测试步骤**：

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 测试空字符串 `""` | 返回错误："此字段为必填项" |
| 2 | 测试只包含空格的字符串 `"   "` | 返回错误："此字段为必填项" |
| 3 | 测试 `null` | 返回错误："此字段为必填项" |
| 4 | 测试 `undefined` | 返回错误："此字段为必填项" |
| 5 | 测试空数组 `[]` | 返回错误："此字段为必填项" |

**非空值应该通过**：

| 输入值 | 预期结果 |
|--------|----------|
| `"test"` | 通过（返回 null） |
| `0` | 通过（返回 null） |
| `false` | 通过（返回 null） |
| `[1, 2, 3]` | 通过（返回 null） |

**自定义错误消息**：
- 调用 `validateRequired("", "请填写此字段")`
- 预期返回："请填写此字段"

---

### 测试 2：email 规则 - 邮箱格式校验

**测试目的**：验证 email 规则能够正确检测邮箱格式。

**测试步骤**：

| 输入值 | 预期结果 |
|--------|----------|
| `"test"` | 返回错误："请输入有效的邮箱地址" |
| `"test@"` | 返回错误："请输入有效的邮箱地址" |
| `"test@.com"` | 返回错误："请输入有效的邮箱地址" |
| `"@example.com"` | 返回错误："请输入有效的邮箱地址" |
| `"test@example.com"` | 通过（返回 null） |
| `"user.name+tag@domain.co.uk"` | 通过（返回 null） |
| `""` | 通过（空值不触发验证） |

---

### 测试 3：maxLength 规则 - 最大长度校验

**测试目的**：验证 maxLength 规则能够正确检测超长文本。

**测试步骤**：

| 输入值 | 规则参数 | 预期结果 |
|--------|----------|----------|
| `"abcdef"` | maxLength: 5 | 返回错误："最多允许 5 个字符" |
| `"1234567890"` | maxLength: 5 | 返回错误："最多允许 5 个字符" |
| `"abcde"` | maxLength: 5 | 通过（刚好等于） |
| `"abc"` | maxLength: 5 | 通过（小于） |
| `""` | maxLength: 5 | 通过（空值不触发） |

**自定义错误消息**：
- 调用 `validateMaxLength("abcdef", 5, "最多 5 个字符")`
- 预期返回："最多 5 个字符"

---

### 测试 4：minLength 规则 - 最小长度校验

**测试目的**：验证 minLength 规则能够正确检测长度不足的文本。

**测试步骤**：

| 输入值 | 规则参数 | 预期结果 |
|--------|----------|----------|
| `"abc"` | minLength: 5 | 返回错误："最少需要 5 个字符" |
| `"abcde"` | minLength: 5 | 通过（刚好等于） |
| `"abcdef"` | minLength: 5 | 通过（超过） |
| `""` | minLength: 5 | 通过（空值不触发） |

---

### 测试 5：自定义校验函数

**测试目的**：验证 custom 规则能够正确执行自定义校验函数。

**测试步骤**：

**场景 A：返回 boolean 值**
```typescript
const validator = (value: string) => value.length > 3;
```

| 输入值 | 预期结果 |
|--------|----------|
| `"abc"` | 返回错误："验证失败" |
| `"abcd"` | 通过（返回 null） |

**场景 B：返回字符串作为错误消息**
```typescript
const validator = (value: string) => {
  if (value.length < 3) {
    return '至少需要 3 个字符';
  }
  return true;
};
```

| 输入值 | 预期结果 |
|--------|----------|
| `"ab"` | 返回："至少需要 3 个字符" |
| `"abcd"` | 通过（返回 null） |

---

### 测试 6：同一字段多条规则按顺序执行

**测试目的**：验证多个验证规则按顺序执行，并收集所有错误。

**测试规则配置**：
```typescript
const rules: ValidationRule[] = [
  { type: 'required', message: '必填' },
  { type: 'minLength', value: 3, message: '最少 3 个字符' },
  { type: 'maxLength', value: 10, message: '最多 10 个字符' },
];
```

**测试步骤**：

| 输入值 | 预期验证结果 | 错误消息 |
|--------|-------------|----------|
| `""` | 失败 | `["必填"]` |
| `"ab"` | 失败 | `["最少 3 个字符"]` |
| `"abcdefghijk"` (11个字符) | 失败 | `["最多 10 个字符"]` |
| `"valid"` | 通过 | `[]` |

**多条规则同时失败的情况**：

```typescript
const rules: ValidationRule[] = [
  { type: 'minLength', value: 5, message: '最少 5 个字符' },
  { type: 'pattern', value: '^[0-9]+$', message: '必须是数字' },
];
```

| 输入值 | 预期验证结果 | 错误消息 |
|--------|-------------|----------|
| `"abc"` | 失败 | `["最少 5 个字符", "必须是数字"]` |

## 第二部分：预览模式表单交互测试

### 测试 1：Input 组件 - 输入文本后值更新

**测试目的**：验证预览模式下 Input 组件可以正常输入并更新值。

**测试环境准备**：
1. 创建一个包含 Form 组件的页面
2. 在 Form 中添加一个 FormItem
3. 在 FormItem 中添加一个 Input 组件
4. 进入预览模式

**测试步骤**：

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 点击 Input 输入框 | 输入框获得焦点，光标闪烁 |
| 2 | 输入文本 "test" | 输入框显示 "test" |
| 3 | 继续输入 "123" | 输入框显示 "test123" |
| 4 | 按 Backspace 键删除字符 | 最后一个字符被删除 |
| 5 | 全选文本后输入新内容 | 原有内容被替换 |

**注意**：预览模式下的输入不会影响编辑模式下的默认值。

### 测试 2：Select 组件 - 选择选项后值更新

## 第三部分：验证视觉反馈测试

### 测试 1：提交时必填字段显示红色错误信息

**测试目的**：验证表单提交时，未通过验证的表单项会显示红色错误信息。

**测试环境准备**：
1. 创建一个 Form 组件，设置 `id` 为 `validationForm`
2. 添加 FormItem 1：
   - 配置 `required: true`
   - 内部添加 Input，配置 validation 规则：`required`（必填）
3. 添加 FormItem 2：
   - 内部添加 Input，配置 validation 规则：`email`（邮箱格式）
4. 添加提交按钮，事件类型："表单提交"，表单 ID：`validationForm`
5. 进入预览模式

**测试步骤**：

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 第一个 Input 保持为空 | - |
| 2 | 第二个 Input 输入 "invalid" | - |
| 3 | 点击提交按钮 | 触发表单验证 |

**预期视觉反馈**：
- 第一个 FormItem（必填字段）：
  - Input 边框变为红色
  - FormItem 下方显示红色错误文字："此字段为必填项"
  - 标签旁边的红色星号（如果有）可能闪烁或保持高亮

- 第二个 FormItem（邮箱字段）：
  - Input 边框变为红色
  - FormItem 下方显示红色错误文字："请输入有效的邮箱地址"

**Console 输出**：
```

  "field1": "",
```

## 第五部分：预览模式与编辑模式隔离测试

### 测试 1：预览模式修改不影响编辑模式

**测试目的**：验证预览模式下的修改不会影响编辑模式下的配置。

**测试步骤**：

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 在编辑模式下，设置 Input 的 `defaultValue` 为 "default" | 配置保存 |
| 2 | 进入预览模式 | Input 显示 "default" |
| 3 | 在预览模式下修改 Input 值为 "modified" | 显示 "modified" |
| 4 | 退出预览模式（返回编辑模式） | Input 的 `defaultValue` 仍然是 "default" |
| 5 | 再次进入预览模式 | Input 显示 "default"（不是 "modified"） |

**结论**：预览模式下的修改是临时的，不会影响编辑模式下的配置。

## 附录：验证规则类型列表

| 规则类型 | 参数 | 默认错误消息 | 说明 |
|----------|------|-------------|------|
| `required` | - | "此字段为必填项" | 必填验证 |
| `minLength` | `value: number` | "最少需要 {n} 个字符" | 最小长度 |
| `maxLength` | `value: number` | "最多允许 {n} 个字符" | 最大长度 |
| `min` | `value: number` | "最小值为 {n}" | 数值最小值 |
| `max` | `value: number` | "最大值为 {n}" | 数值最大值 |
| `pattern` | `value: string \| RegExp` | "格式不正确" | 正则匹配 |
| `email` | - | "请输入有效的邮箱地址" | 邮箱格式 |
| `url` | - | "请输入有效的URL" | URL 格式 |
| `custom` | `customValidator: Function` | "验证失败" | 自定义函数 |
