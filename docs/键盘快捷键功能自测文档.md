# 键盘快捷键功能自测文档

## 一、功能概述

键盘快捷键功能为低代码编辑器提供了快速操作能力，提升开发效率。该功能支持：

| 快捷键 | Windows/Linux | macOS | 功能 |
|-------|--------------|-------|-----|
| 删除组件 | `Delete` / `Backspace` | `Delete` / `Backspace` | 删除当前选中的组件 |
| 撤销 | `Ctrl + Z` | `⌘ + Z` | 撤销上一步操作 |
| 重做 | `Ctrl + Y` / `Ctrl + Shift + Z` | `⌘ + Y` / `⌘ + ⇧ + Z` | 重做下一步操作 |

### 关键特性

1. **路由隔离**：仅在 `/builder` 路由下生效
2. **焦点冲突处理**：输入框中不触发快捷键
3. **跨平台兼容**：自动检测 Windows/Linux/macOS 并使用正确的修饰键
4. **UI 提示**：在 Header 中显示快捷键标签

## 三、详细测试步骤

### 测试 1：Delete 键删除选中组件

**目的**：验证选中组件后按 Delete 键能正确删除组件

**前置条件**：
- 在 `/builder` 页面
- 画布上至少有一个组件

**步骤**：
1. 点击画布上的一个组件（确保组件被选中，有高亮边框）
2. 观察 store 中的 `selectedComponentId` 应该等于该组件的 ID
3. 按键盘上的 `Delete` 键（或笔记本上的 `Backspace` 键）

**预期结果**：
- ✅ 组件从画布上消失
- ✅ 组件从 store 的 `components` 数组中被移除
- ✅ `selectedComponentId` 变为 `null`
- ✅ 该操作应该被记录到历史记录（可以按 Ctrl+Z 撤销）

**异常场景**：
- ❌ 组件没有被删除 → 检查 `handleDelete` 回调是否正确触发
- ❌ 组件被删除但 selectedComponentId 没有清空 → 检查 `removeComponent` 方法

## 五、核心逻辑说明

### 1. 事件监听机制

```typescript
useEffect(() => {
  window.addEventListener('keydown', handleKeyDown, { capture: true });
  
  return () => {
    window.removeEventListener('keydown', handleKeyDown, { capture: true });
  };
}, [handleKeyDown]);
```

**关键点**：
- 使用 `capture: true` 确保在事件捕获阶段就处理
- 这样可以在输入框的默认行为之前进行判断
- 但只有当条件满足时才调用 `event.preventDefault()`

### 2. 路由检测逻辑

```typescript
const isBuilderRoute = (pathname: string): boolean => {
  return pathname === '/builder' || pathname.startsWith('/builder/');
};
```

**关键点**：
- 精确匹配 `/builder`
- 前缀匹配 `/builder/`（支持子路由）
- 使用 `useLocation()` Hook 获取当前路径

### 3. 输入元素检测逻辑

```typescript
const isInputElement = (element: HTMLElement | null): boolean => {
  if (!element) return false;
  
  const tagName = element.tagName.toLowerCase();
  if (tagName === 'input' || tagName === 'textarea' || tagName === 'select') {
    return true;
  }
  
  if (element.isContentEditable) {
    return true;
  }
  
  return false;
};
```

**关键点**：
- 检测标准表单元素：`<input>`, `<textarea>`, `<select>`
- 检测 `contentEditable` 属性（富文本编辑器）
- 从 `document.activeElement` 获取当前焦点元素

### 4. 修饰键检测逻辑

```typescript
const isPrimaryModifierKey = (event: KeyboardEvent): boolean => {
  return isApplePlatform() ? event.metaKey : event.ctrlKey;
};
```

**关键点**：
- Apple 平台（macOS/iOS）使用 `metaKey`（Command 键）
- Windows/Linux 使用 `ctrlKey`（Control 键）
- 自动检测平台并选择正确的修饰键

## 六、常见问题排查

### 问题 1：快捷键完全没有反应

**可能原因**：
1. 不在 `/builder` 路由下
2. 焦点在输入框上
3. Hook 没有正确启用

**排查步骤**：
1. 检查 `location.pathname` 是否为 `/builder`
2. 检查 `document.activeElement` 的标签名
3. 检查 `App.tsx` 中是否正确调用了 `useKeyboardShortcuts()`

### 问题 2：输入框中按 Delete 也删除了组件

**可能原因**：
1. `isInputElement` 函数有 bug
2. 事件监听没有使用 `capture: true`
3. 条件判断顺序错误

**排查步骤**：
1. 测试 `isInputElement(document.createElement('input'))` 是否返回 `true`
2. 检查 `addEventListener` 的第三个参数
3. 确认 `isInputElement` 判断在快捷键处理之前

### 问题 3：Mac 上按 Cmd+Z 没有反应

**可能原因**：
1. 平台检测错误
2. 事件没有正确捕获

**排查步骤**：
1. 在控制台打印 `navigator.platform`
2. 测试 `isMac()` 返回值
3. 检查 `event.metaKey` 是否为 `true`

### 问题 4：Header 中没有显示快捷键标签

**可能原因**：
1. 屏幕宽度不够（响应式隐藏）
2. CSS 类名错误
3. 导入路径错误

**排查步骤**：
1. 检查屏幕宽度是否大于 lg 断点（通常 1024px）
2. 检查 `getKeyboardShortcutsInfo` 函数是否正确导入
3. 检查控制台是否有导入错误
