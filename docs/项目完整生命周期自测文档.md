# 项目完整生命周期自测文档

## 文档概述

本文档以用户操作路径的形式，描述项目完整生命周期的测试用例。涵盖：
- 项目创建、保存、加载、切换、删除
- 跨页面导航状态一致性
- 异常场景处理
- 导入导出功能
- 浏览器后退/前进导航

---

## 测试环境准备

### 前置条件

1. **清空测试数据**
   - 打开开发者工具 → Application → Local Storage
   - 删除所有以 `lowcode_builder_project_` 开头的 key
   - 删除 `lowcode_builder_project_list`

2. **浏览器环境**
   - 建议使用 Chrome/Edge 浏览器
   - 打开开发者工具（F12）
   - 切换到 Console 标签页，监控错误日志
   - 切换到 Network 标签页（可选）

3. **启动应用**
   - 运行 `npm run dev` 启动开发服务器
   - 访问 `http://localhost:5173`

## 第一部分：项目完整生命周期测试

### 场景 1：新建项目 → 保存 → 刷新页面 → 自动恢复

**测试目的**：验证项目从创建到自动恢复的完整流程

#### 操作步骤

**步骤 1：访问应用**
- 操作：在浏览器访问 `http://localhost:5173`
- 预期：
  - URL 自动重定向到 `/projects`
  - 显示「还没有项目」的空状态页面
  - 控制台无错误

**步骤 2：新建项目**
- 操作：点击「新建项目」按钮
- 预期：
  - 导航到 `/builder` 编辑器页面
  - 显示空画布
  - 顶部项目名称显示为「项目 1」（或类似名称）
  - 控制台无错误

**步骤 3：添加组件**
- 操作：
  1. 从左侧组件库拖拽「文本」组件到画布
  2. 从左侧组件库拖拽「按钮」组件到画布
  3. 从左侧组件库拖拽「容器」组件到画布
- 预期：
  - 画布上显示 3 个组件
  - 组件可选中、可移动
  - 控制台无错误

**步骤 4：等待自动保存**
- 操作：
  1. 观察右上角保存状态
  2. 等待显示「已保存」或状态变为 `saved`
  3. 或手动点击「保存」按钮
- 预期：
  - 保存状态显示正确
  - 控制台无错误
  - Local Storage 中出现新项目数据

**步骤 5：验证存储**
- 操作：
  1. 打开开发者工具 → Application → Local Storage
  2. 查看以 `lowcode_builder_project_` 开头的 key
- 预期：
  - 存在一个新项目 key（如 `lowcode_builder_project_project_xxx`）
  - `lowcode_builder_project_list` 包含新项目 ID

**步骤 6：刷新页面**
- 操作：按 F5 或点击浏览器刷新按钮
- 预期：
  - 页面正常加载，无白色错误页
  - 控制台无 `Maximum update depth exceeded` 错误
  - 无任何红色错误日志

**步骤 7：验证自动恢复**
- 操作：
  1. 观察画布上的组件
  2. 查看顶部项目名称
  3. 检查控制台
- 预期：
  - 画布上仍有 3 个组件（文本、按钮、容器）
  - 项目名称与刷新前一致
  - 控制台无错误
  - 出现 Toast 提示：「已恢复上次编辑的项目」

**步骤 8：验证可编辑性**
- 操作：
  1. 选中一个组件
  2. 尝试移动它
  3. 尝试添加新组件
- 预期：
  - 组件可选中
  - 组件可移动
  - 可添加新组件
  - 控制台无错误

### 场景 2：切换到项目列表 → 打开另一个项目

**测试目的**：验证从项目列表切换到不同项目的流程

#### 操作步骤

**步骤 1：创建多个项目**
- 操作：
  1. 在编辑器中添加一些组件
  2. 点击顶部项目名称，导航到 `/projects`
  3. 点击「新建项目」，创建第二个项目
  4. 重复创建第三个项目
- 预期：
  - 项目列表显示 3 个项目
  - 最新创建的项目显示「当前打开」标记
  - 控制台无错误

**步骤 2：从列表打开项目**
- 操作：
  1. 选择**不是**「当前打开」的项目
  2. 点击项目卡片上的「打开」按钮
  3. 或点击项目卡片主体
- 预期：
  - 导航到 `/builder` 编辑器页面
  - 画布显示的是点击的项目的组件
  - 顶部项目名称是点击的项目名称
  - 控制台无错误
  - 无无限循环
  - 无白色错误页

**步骤 3：验证状态一致性**
- 操作：
  1. 查看组件数量和类型
  2. 尝试添加新组件
  3. 尝试保存
- 预期：
  - 组件与创建时一致
  - 可正常编辑
  - 可正常保存
  - 控制台无错误

### 场景 3：删除项目 → 确认跳转

**测试目的**：验证删除项目后的行为

#### 操作步骤

**步骤 1：准备测试项目**
- 操作：
  1. 创建项目 A
  2. 创建项目 B
  3. 确保当前打开的是项目 B
- 预期：
  - 项目列表有 2 个项目
  - 项目 B 显示「当前打开」标记

**步骤 2：删除当前项目**
- 操作：
  1. 导航到 `/projects`
  2. 找到项目 B（当前打开的项目）
  3. 点击项目卡片右上角的「删除」按钮（垃圾桶图标）
  4. 在确认弹窗中点击「删除」
- 预期：
  - 项目 B 从列表中消失
  - 项目 A 仍然在列表中
  - 如果代码中有自动跳转逻辑，应该跳转到某个页面
  - 控制台无错误

**步骤 3：验证已删除项目无法加载**
- 操作：
  1. 打开项目 A
  2. 尝试通过某种方式加载已删除的项目 B（如果有测试方法）
- 预期：
  - 已删除项目无法加载
  - 如果尝试加载，应该显示友好的错误提示
  - 不会崩溃

**步骤 4：验证剩余项目正常**
- 操作：
  1. 打开项目 A
  2. 尝试编辑和保存
- 预期：
  - 项目 A 正常工作
  - 可编辑、可保存
  - 控制台无错误

## 第四部分：导入导出功能测试

### 场景 9：导出当前加载的项目数据

**测试目的**：验证导出功能使用正确的项目数据

#### 操作步骤

**步骤 1：准备测试项目**
- 操作：
  1. 新建项目「导出测试项目」
  2. 添加组件：
     - 文本组件「导出文本」
     - 按钮组件「导出按钮」
     - 容器组件，内部添加一个嵌套文本
  3. 保存
- 预期：
  - 画布有 3 个顶层组件
  - 容器内有 1 个嵌套组件

**步骤 2：执行导出**
- 操作：
  1. 点击顶部「导出」按钮
  2. 观察下载的文件
- 预期：
  - 下载一个 JSON 文件
  - 文件名包含项目名称，如 `导出测试项目_xxx.json`
  - 控制台无错误

**步骤 3：验证导出文件内容**
- 操作：
  1. 打开下载的 JSON 文件
  2. 检查内容
- 预期：
  - JSON 结构正确
  - `name` 字段是「导出测试项目」
  - `components` 数组包含 3 个顶层组件
  - 容器组件的 `children` 包含嵌套文本
  - 所有组件 ID 与画布上的一致

**步骤 4：验证导出的是当前项目**
- 操作：
  1. 切换到另一个项目 B
  2. 点击「导出」
- 预期：
  - 导出的是项目 B 的数据
  - 不是之前项目的数据
  - 每次导出都使用当前加载的项目状态
