# 事件交互系统自测文档

## 文档概述

本文档记录了事件交互系统的测试步骤、预期结果和测试矩阵。事件系统支持：

- **事件类型**：onClick、onChange、onSubmit、onFocus、onBlur
- **动作类型**：SHOW_ALERT、NAVIGATE_URL、NAVIGATE_PAGE、CONSOLE_LOG、CUSTOM_SCRIPT、FORM_SUBMIT、FORM_RESET
- **特性**：多动作顺序执行、动作启用/禁用、未知动作静默忽略、异常捕获

---

**测试步骤**：

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 调用 `executeActions(actions)` | 不抛出错误到外部 |
| 2 | 检查 console 输出 | 输出 "动作 1"、错误信息、"动作 3" |
| 3 | 检查 alert | 显示 "脚本执行错误: Error: 故意抛出错误" |
| 4 | 验证动作 3 是否执行 | 动作 3 仍然执行，证明错误被捕获 |

## 第七部分：常见问题排查

### 问题 1：事件配置保存后加载不完整

**可能原因**：
- 数据结构不支持 JSON 序列化
- 枚举值在序列化时丢失

**排查步骤**：
1. 检查 localStorage 中的原始数据
2. 确认所有字段都是可序列化的（没有函数、Symbol 等）
3. 检查枚举值是否以字符串形式保存

### 问题 2：预览模式下事件不触发

**可能原因**：
- 忘记切换到预览模式
- 动作全部 disabled
- 事件配置错误

**排查步骤**：
1. 确认处于预览模式（不是编辑模式）
2. 检查动作的 enabled 状态
3. 检查组件是否为可交互组件
4. 检查事件配置是否关联到正确的事件类型

### 问题 3：自定义脚本执行错误

**可能原因**：
- 脚本语法错误
- 脚本运行时错误

**排查步骤**：
1. 在浏览器控制台单独测试脚本
2. 使用简单的 `console.log` 验证脚本是否执行
3. 检查脚本中是否使用了未定义的变量

### 问题 4：NAVIGATE_URL 不跳转

**可能原因**：
- 缺少 targetUrl 参数
- 浏览器阻止弹窗（新窗口打开时）
- URL 格式错误

**排查步骤**：
1. 检查 targetUrl 是否正确配置
2. 检查浏览器是否阻止了弹窗
3. 确认 URL 包含协议（http:// 或 https://）

## 附录：数据结构参考

### ActionConfig（动作配置）
```typescript
interface ActionConfig {
  id: string;           // 唯一标识符
  type: ActionType;     // 动作类型枚举
  params: {             // 动作参数
    alertMessage?: string;
    targetUrl?: string;
    navigateTarget?: NavigateTarget;
    pageId?: string;
    logMessage?: string;
    customScript?: string;
    formId?: string;
  };
  enabled: boolean;     // 是否启用
}
```

### EventConfig（事件配置）
```typescript
interface EventConfig {
  type: EventType;      // 事件类型
  actions: ActionConfig[];  // 动作列表
  enabled: boolean;     // 事件是否启用
}
```

### ComponentEvents（组件事件）
```typescript
interface ComponentEvents {
  onClick?: ClickEventConfig;          // 旧版点击事件（向后兼容）
  onClickActions?: EventConfig;        // 新版点击事件
  onChangeActions?: EventConfig;       // 值变更事件
  onSubmitActions?: EventConfig;       // 表单提交事件
  onFocusActions?: EventConfig;        // 获得焦点事件
  onBlurActions?: EventConfig;         // 失去焦点事件
}
```
