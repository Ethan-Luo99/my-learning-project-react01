# 高级业务组件自测文档

## 一、功能概述

本文档描述了 5 个高级业务组件的开发和测试要点：

| 组件 | 类型 | 主要功能 | 容器组件 |
|-----|------|---------|---------|
| **Card** | 布局组件 | 卡片式容器，支持 header/body/footer 三个区域 | ✅ 是 |
| **Divider** | 布局组件 | 水平/垂直分割线，支持中间文字 | ❌ 否 |
| **Tabs** | 布局组件 | 标签页，包含多个 TabPane 子面板 | ✅ 是 |
| **TabPane** | 布局组件 | 标签面板，属于 Tabs 的子组件 | ✅ 是 |
| **Accordion** | 布局组件 | 折叠面板，包含多个 AccordionItem | ✅ 是 |
| **AccordionItem** | 布局组件 | 折叠项，属于 Accordion 的子组件 | ✅ 是 |
| **Modal** | 布局/交互组件 | 弹窗组件，支持事件触发打开/关闭 | ✅ 是 |

---

## 二、组件详情

### 2.1 Card 卡片组件

#### 属性配置

| 属性名 | 类型 | 默认值 | 说明 |
|-------|------|--------|------|
| `shadow` | `'none' \| 'sm' \| 'md' \| 'lg'` | `'md'` | 阴影大小 |
| `padding` | `'none' \| 'sm' \| 'md' \| 'lg'` | `'md'` | 内边距大小 |
| `bordered` | `boolean` | `true` | 是否显示边框 |
| `headerTitle` | `string` | `'卡片标题'` | 头部标题 |
| `hoverable` | `boolean` | `false` | 鼠标悬停时是否有效果 |

#### 默认配置

```typescript
{
  type: ComponentType.Card,
  label: '卡片',
  defaultWidth: 350,
  defaultHeight: 'auto',
  defaultProps: {
    shadow: 'md',
    padding: 'md',
    bordered: true,
    headerTitle: '卡片标题',
    hoverable: false,
  },
  defaultStyles: {
    minHeight: '150px',
  },
}
```

#### 测试要点

1. **基础渲染**
   - ✅ 组件类型为 `ComponentType.Card`
   - ✅ 默认宽度为 350px
   - ✅ 默认有边框 `bordered: true`
   - ✅ 默认阴影 `shadow: 'md'`
   - ✅ 默认有头部标题

2. **子组件支持**
   - ✅ 支持嵌套子组件（`children` 属性）
   - ✅ 支持多层嵌套（Card 内嵌套 Card）
   - ✅ 可拖拽其他组件进入 Card 内部

3. **自定义属性**
   - ✅ 可设置 `shadow` 为 `'none'`、`'sm'`、`'md'`、`'lg'`
   - ✅ 可设置 `bordered: false` 隐藏边框
   - ✅ 可设置 `hoverable: true` 开启悬停效果
   - ✅ 可自定义 `headerTitle`

## 三、事件系统集成

### 3.1 SHOW_MODAL 动作

**功能**：通过事件触发打开指定的 Modal

#### 参数

| 参数 | 类型 | 必需 | 说明 |
|-----|------|------|------|
| `modalId` | `string` | ✅ 是 | 目标 Modal 组件的 ID |

#### 执行逻辑

```
SHOW_MODAL(modalId)
  ├── 如果缺少 modalId → 输出警告
  ├── 如果缺少上下文 openModal → 输出警告
  └── 调用 context.openModal(modalId)
```

### 3.2 HIDE_MODAL 动作

**功能**：通过事件触发关闭指定的 Modal

#### 参数

| 参数 | 类型 | 必需 | 说明 |
|-----|------|------|------|
| `modalId` | `string` | ✅ 是 | 目标 Modal 组件的 ID |

#### 执行逻辑

```
HIDE_MODAL(modalId)
  ├── 如果缺少 modalId → 输出警告
  ├── 如果缺少上下文 closeModal → 输出警告
  └── 调用 context.closeModal(modalId)
```

### 3.3 Modal Registry 上下文

`PreviewModalRegistry` 上下文管理所有 Modal 的显示/隐藏状态：

#### 接口

```typescript
interface ModalRegistryContextValue {
  visibleModals: Set<string>;
  openModal: (modalId: string) => void;
  closeModal: (modalId: string) => void;
  toggleModal: (modalId: string) => void;
  isModalVisible: (modalId: string) => boolean;
}
```

#### 使用场景

1. **按钮点击打开弹窗**
   - Button 配置 `onClickActions`
   - 添加 `SHOW_MODAL` 动作，设置 `modalId`
   - 预览模式下点击按钮 → 打开 Modal

2. **弹窗确定/取消按钮**
   - Modal 的确定按钮可配置 `onOkActions`
   - Modal 的取消按钮可配置 `onCancelActions`
   - 常见配置：添加 `HIDE_MODAL` 动作关闭自身

## 八、常见问题排查

### 问题 1：Modal 在编辑模式下无法拖入子组件

**可能原因**：
- Modal 未被识别为容器组件
- `isContainerComponent` 函数未包含 Modal

**排查步骤**：
1. 检查 `src/components/builder/DndContext.tsx` 中的 `isContainerComponent`
2. 检查 `src/store/useBuilderStore.ts` 中的 `isContainerComponent`
3. 确认 `ComponentType.Modal` 在判断条件中

### 问题 2：SHOW_MODAL 动作不生效

**可能原因**：
- `modalId` 配置错误
- 事件引擎未正确执行 `SHOW_MODAL`
- Modal Registry 上下文未提供

**排查步骤**：
1. 检查 Button 的 `onClickActions` 配置
2. 确认 `modalId` 与目标 Modal 的 `id` 一致
3. 检查预览模式下 `PreviewModalRegistryProvider` 是否正确包裹
4. 检查 `eventEngine.ts` 中的 `executeShowModal` 实现

### 问题 3：属性面板修改后画布不更新

**可能原因**：
- Zustand 状态更新未触发重渲染
- 组件渲染逻辑未正确响应 props 变化

**排查步骤**：
1. 检查 `useBuilderStore` 中的 `updateComponentProps` 方法
2. 检查组件渲染时是否使用了最新的 props
3. 确认没有不必要的 `React.memo` 阻止更新

### 问题 4：Tabs/Accordion 子组件不显示

**可能原因**：
- 子组件没有被正确识别为 TabPane/AccordionItem
- 渲染逻辑中的类型判断错误

**排查步骤**：
1. 检查 `ComponentRenderer/index.tsx` 中的 `renderTabs` 和 `renderAccordion`
2. 确认子组件的 `type` 为 `ComponentType.TabPane` 或 `ComponentType.AccordionItem`
3. 检查数据结构中 `children` 数组是否正确
