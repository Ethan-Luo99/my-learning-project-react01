# 项目加载无限循环 Bug 事后分析文档

**日期**: 2026-05-03  
**严重程度**: 高（阻塞核心功能）  
**影响范围**: 所有用户无法进入编辑器

---

## 1. Bug 现象描述

### 1.1 用户可见效果

用户在以下场景会遇到问题：

1. **从项目列表打开项目**
   - 操作：在 `/projects` 页面点击项目卡片上的「继续编辑」或「打开」按钮
   - 效果：页面瞬间被白色错误页覆盖，无法进入编辑器

2. **直接访问编辑器**
   - 操作：直接访问 `/builder` 页面
   - 效果：如果有已保存的项目，同样出现白色错误页

3. **刷新编辑器页面**
   - 操作：在编辑器中按 F5 刷新
   - 效果：出现白色错误页

### 1.2 控制台错误

浏览器控制台显示以下错误：

```
Error: Maximum update depth exceeded. This can happen when a component repeatedly calls setState inside componentWillUpdate or componentDidUpdate. React limits the number of nested updates to prevent infinite loops.
```

**错误栈**：
```
at checkForNestedUpdates
    at scheduleUpdateOnFiber
    at setComponents
    at loadLatestProject
    at App.tsx:118
    at commitHookEffectListMount
    at commitLayoutEffectOnFiber
    ... (无限重复)
```

### 1.3 技术特征

| 特征 | 描述 |
|-----|------|
| 触发条件 | 调用 `loadLatestProject` 或 `loadProject` 后 |
| 影响用户 | 所有用户，无论项目数量 |
| 复现率 | 100%（有项目时） |
| 自行恢复 | 否，必须修复代码 |

## 2. 根因分析

### 2.1 闭环链路图

```
┌─────────────────────────────────────────────────────────────────┐
│                        无限循环触发链路                            │
└─────────────────────────────────────────────────────────────────┘

                           ┌──────────────┐
                           │  初始挂载     │
                           └──────┬───────┘
                                  │
                                  ▼
                    ┌─────────────────────────┐
                    │ App.tsx  useEffect 执行 │
                    │ 依赖: [loadLatestProject,│
                    │       toast]            │
                    └───────────┬─────────────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │ 调用 loadLatestProject  │
                    │ (从 store 中获取的函数)  │
                    └───────────┬─────────────┘
                                │
                                ▼
              ┌───────────────────────────────────────┐
              │    useBuilderStore.loadLatestProject   │
              │                                         │
              │   1. getLatestProject() 获取数据       │
              │   2. setComponents(components)    ❌   │
              │      → 调用 pushHistory()              │
              │      → 更新 history, currentIndex,     │
              │        canUndo, canRedo, components   │
              │   3. set({...}) 再次更新状态      ❌   │
              └───────────────────┬───────────────────┘
                                  │
                                  ▼
                    ┌─────────────────────────┐
                    │   Zustand 状态变化       │
                    │   函数引用变化           │
                    └───────────┬─────────────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │ useEffect 依赖项变化     │
                    │ loadLatestProject 引用   │
                    │ 发生变化                 │
                    └───────────┬─────────────┘
                                │
                                ▼
              ┌───────────────────────────────────────┐
              │         useEffect 重新执行             │
              │                                         │
              │  问题：如果依赖项中有函数引用，         │
              │  每次状态更新都可能导致引用变化，        │
              │  从而触发 useEffect 重新执行            │
              └───────────────────┬───────────────────┘
                                  │
                                  ▼
                    ┌─────────────────────────┐
                    │   回到调用 loadLatestProject  │
                    │   ⟲ 开始无限循环       │
                    └─────────────────────────┘
```

### 2.2 详细分析

#### 问题 1：Store 层 - 多次状态更新

**原代码 (`loadLatestProject`)**：

```typescript
loadLatestProject: () => {
  const project = getLatestProject();
  if (!project) {
    return false;
  }

  const { setComponents } = get();
  setComponents(project.components);  // ❌ 第一次 set()

  set(
    {
      currentProjectId: project.id,
      projectName: project.name,
      // ...
    },
    false,
    'loadLatestProject'  // ❌ 第二次 set()
  );

  return true;
};
```

**`setComponents` 的实现**：

```typescript
setComponents: (newComponents) => {
  const newHistoryEntry: HistoryState = {
    components: structuredClone(newComponents),
    selectedComponentId: state.selectedComponentId,
  };

  const newHistory = state.history.slice(0, state.currentIndex + 1);
  newHistory.push(newHistoryEntry);  // ❌ 推入历史记录

  if (newHistory.length > MAX_HISTORY_SIZE) {
    newHistory.shift();
  }

  set({
    components: newComponents,
    history: newHistory,
    currentIndex: newHistory.length - 1,
    canUndo: newHistory.length > 1,
    canRedo: false,
  }, false, 'setComponents');  // ❌ 第一次 set()
},
```

**问题分析**：

| 调用顺序 | 方法 | 更新的状态 | 问题 |
|---------|------|-----------|------|
| 1 | `setComponents` | `components`, `history`, `currentIndex`, `canUndo`, `canRedo` | 推入历史记录，应该只在编辑操作时调用 |
| 2 | `set` 第二次 | `currentProjectId`, `projectName`, `lastSavedAt`, ... | 多余的状态更新 |

#### 问题 2：组件层 - 不稳定的依赖项

**原代码 (`App.tsx`)**：

```typescript
const loadLatestProject = useBuilderStore((state) => state.loadLatestProject);

useEffect(() => {
  const hasProject = loadLatestProject();
  if (hasProject) {
    toast.info('已恢复上次编辑的项目');
  }
}, [loadLatestProject, toast]);  // ❌ 依赖函数引用
```

**问题分析**：

在 Zustand 中，虽然通过 `(state) => state.method` 获取的函数引用通常是稳定的，但如果：

1. **store 内部状态变化** → 可能影响函数闭包
2. **store 中间件** → 可能改变函数引用
3. **Zustand 版本差异** → 行为可能不一致

**核心问题**：

`useEffect` 的依赖项应该是**值**，而不是**可变的函数引用**。即使函数逻辑不变，如果引用发生变化，`useEffect` 也会重新执行。

#### 问题 3：缺少防御性检查

**原代码 (`loadLatestProject`)**：

```typescript
loadLatestProject: () => {
  const project = getLatestProject();
  if (!project) {
    return false;
  }
  // ... 直接加载
};
```

**缺少的检查**：

```typescript
// 应该检查当前是否已有项目
loadLatestProject: () => {
  const { currentProjectId } = get();  // ✅ 获取当前项目 ID
  
  if (currentProjectId) {
    return false;  // ✅ 已有项目，跳过自动恢复
  }
  // ... 加载项目
};
```

### 2.3 问题根源总结

| 层面 | 问题 | 根本原因 |
|-----|------|---------|
| **Store 层** | 多次 `set()` 调用 + `pushHistory` | 加载项目时使用了编辑操作的方法 |
| **组件层** | 依赖函数引用 | 违反了 `useEffect` 依赖项的最佳实践 |
| **防御层** | 缺少状态检查 | 没有利用 `currentProjectId` 避免重复加载 |

## 3. 修复方案

### 3.1 修复原则

修复遵循以下三个原则：

1. **单次状态更新**：加载操作使用单个 `set()` 原子更新所有状态
2. **历史重置**：加载项目时重置历史记录，不推入新记录
3. **空依赖数组**：`useEffect` 使用 `[]` 配合 `useRef` 实现"只执行一次"

### 3.2 Store 层修复

#### 修复 1：`loadLatestProject`

**修改前**：
```typescript
loadLatestProject: () => {
  const project = getLatestProject();
  if (!project) {
    return false;
  }

  const { setComponents } = get();
  setComponents(project.components);  // ❌ 推入历史

  set(
    {
      currentProjectId: project.id,
      projectName: project.name,
      // ...
    },
    false,
    'loadLatestProject'
  );

  return true;
};
```

**修改后**：
```typescript
loadLatestProject: () => {
  const { currentProjectId } = get();  // ✅ 新增检查
  
  if (currentProjectId) {
    return false;  // ✅ 已有项目，跳过
  }

  const project = getLatestProject();
  if (!project) {
    return false;
  }

  const initialHistory: HistoryState[] = [
    {
      components: structuredClone(project.components),
      selectedComponentId: null,
    },
  ];

  set(
    {
      components: project.components,
      currentProjectId: project.id,
      projectName: project.name,
      lastSavedAt: project.updatedAt,
      selectedComponentId: null,
      saveStatus: 'idle',
      saveErrorMessage: null,
      loadError: null,
      isProjectCorrupted: false,
      history: initialHistory,  // ✅ 重置历史
      currentIndex: 0,
      canUndo: false,
      canRedo: false,
    },
    false,
    'loadLatestProject'
  );

  return true;
};
```

#### 修复 2：`loadProject`

与 `loadLatestProject` 类似，修改为单次 `set()` + 重置历史。

#### 修复 3：`saveCurrentAndCreateNewProject`

**修改前**：

### 3.3 组件层修复

#### 修复 1：加载项目 `useEffect`

**修改前**：
```typescript

const hasLoadedInitialProject = useRef(false);

    return;
```

#### 修复 2：加载错误 `useEffect`

**修改前**：
```typescript
const clearLoadError = useBuilderStore((state) => state.clearLoadError);

useEffect(() => {
  if (loadError) {
    // ...
```

```typescript
// 不再从 store 中获取 clearLoadError
```

## 4. 同类模式排查

### 4.1 排查范围

排查了以下所有涉及 `set()` 或状态更新的 store 方法：

| 方法名 | 调用 `set()` 次数 | 调用 `pushHistory` | 风险等级 |
|-------|-----------------|-------------------|---------|
| `loadLatestProject` | 2 次 | 是（通过 `setComponents`） | ❌ 高风险 |
| `loadProject` | 2 次 | 是（通过 `setComponents`） | ❌ 高风险 |
| `createNewProject` | 1 次 | 否 | ✅ 低风险 |
| `saveCurrentAndLoadProject` | 1 次 (`loadProject`) | 是（通过 `loadProject`） | ⚠️ 中等（依赖 `loadProject`） |
| `saveCurrentAndCreateNewProject` | 2 次 | 否 | ⚠️ 中等（多次 `set()`） |

#### 已修复的方法

| 方法 | 修复内容 |

| `loadLatestProject` | 添加 `currentProjectId` 检查 + 单次 `set()` + 重置 `history` |

| `saveCurrentAndCreateNewProject` | 合并为单次 `set()` + 重置 `history` |

#### 无需修改的方法

| 方法 | 结论 | 原因 |
|-----|------|------|

## 5. Zustand + React useEffect 编码规范

基于本次 Bug 的经验教训，总结以下三条编码规范：

### 规范 1：Store 方法状态更新原则

**规则**：加载/初始化类方法使用单次 `set()`，编辑类方法可多次更新。

**详细说明**：

| 方法类型 | 场景 | 状态更新策略 | 示例 |
|---------|------|-------------|------|
| **加载类** | 加载项目、恢复数据、初始化 | 单次 `set()`，重置 `history` | `loadLatestProject`, `loadProject` |
| **创建类** | 新建项目、清空画布 | 单次 `set()`，重置 `history` | `createNewProject` |
| **编辑类** | 添加组件、删除组件、修改属性 | 可多次 `set()`，推入 `history` | `addComponent`, `removeComponent` |

**加载类方法标准模板**：

```typescript
loadXXX: () => {
  const { currentProjectId } = get();  // 1. 检查当前状态
  
  if (currentProjectId) {
    return false;  // 2. 已有则跳过
  }

  const data = loadDataFromStorage();
  if (!data) {
    return false;
  }

  const initialHistory: HistoryState[] = [  // 3. 初始化历史
    {
      components: structuredClone(data.components),
      selectedComponentId: null,
    },
  ];

  set(
    {
      components: data.components,
      currentProjectId: data.id,
      projectName: data.name,
      // ... 其他状态
      history: initialHistory,  // 4. 重置历史
      currentIndex: 0,
      canUndo: false,
      canRedo: false,
    },
    false,
    'loadXXX'  // 5. 单次 set()
  );

  return true;
};
```

**禁止的模式**：

```typescript
// ❌ 禁止：加载方法中调用 setComponents（会推入历史）
loadProject: () => {
  const { setComponents } = get();
  setComponents(components);  // ❌ 推入历史，应该用于编辑操作
  set({ ... });  // ❌ 多次 set()
};

// ❌ 禁止：加载方法中使用 addComponent/removeComponent 等
loadProject: () => {
  components.forEach(c => addComponent(c));  // ❌ 每次调用都会推入历史
};
```

---

### 规范 2：useEffect 依赖项选取规则

**规则**：`useEffect` 依赖项优先使用状态值，避免依赖函数引用。

**详细说明**：

| 依赖项类型 | 安全性 | 说明 |
|-----------|--------|------|
| **原始值** | ✅ 安全 | `string`, `number`, `boolean`, `null`, `undefined` |
| **对象/数组** | ⚠️ 需注意 | 如果是新创建的对象/数组，每次渲染都是新引用 |
| **状态值** | ✅ 安全 | 从 store 或 useState 获取的状态值 |
| **函数引用** | ❌ 避免 | 函数引用可能不稳定 |
| **方法引用** | ❌ 避免 | 从 store 获取的方法引用 |

**正确的模式**：

**模式 A：空依赖数组 + useRef**（推荐用于"只执行一次"的场景）

```typescript
const hasExecuted = useRef(false);

useEffect(() => {
  if (hasExecuted.current) {
    return;
  }
  hasExecuted.current = true;
  
  // 执行初始化逻辑
  loadLatestProject();
}, []);  // ✅ 空依赖数组
```

**模式 B：依赖状态值 + 条件检查**

```typescript
const loadError = useBuilderStore((state) => state.loadError);
const currentProjectId = useBuilderStore((state) => state.currentProjectId);

useEffect(() => {
  if (loadError) {
    // 处理错误
    useBuilderStore.getState().clearLoadError();  // ✅ 直接调用，不依赖引用
  }
}, [loadError, currentProjectId]);  // ✅ 依赖状态值
```

**模式 C：useCallback 包装（当必须依赖函数时）**

```typescript
const someValue = useBuilderStore((state) => state.someValue);

const stableFunction = useCallback(() => {
  const state = useBuilderStore.getState();
  state.someMethod(someValue);
}, [someValue]);  // ✅ 依赖状态值

useEffect(() => {
  stableFunction();
}, [stableFunction]);  // ✅ 依赖 useCallback 包装的函数
```

**禁止的模式**：

```typescript
// ❌ 禁止：依赖从 store 获取的方法引用
const loadLatestProject = useBuilderStore((state) => state.loadLatestProject);

useEffect(() => {
  loadLatestProject();
}, [loadLatestProject]);  // ❌ 函数引用可能变化

// ❌ 禁止：在 useEffect 内创建新函数作为依赖
useEffect(() => {
  const fetchData = () => { /* ... */ };
  fetchData();
}, [fetchData]);  // ❌ 每次渲染 fetchData 都是新引用
```

### 规范 3：组件挂载时调用 Store 方法的安全模式

**规则**：组件挂载时调用 store 方法必须有双重保护机制。

**双重保护机制**：

```
┌─────────────────────────────────────────────────────────────┐
│                      双重保护机制                             │
└─────────────────────────────────────────────────────────────┘

                         组件挂载
                             │
                             ▼
              ┌──────────────────────────────┐
              │   第一层：useRef 标记         │
              │   hasLoadedInitialProject    │
              └──────────────┬───────────────┘
                             │
              ┌──────────────┴───────────────┐
              │   if (hasLoaded.current)     │
              │       return;                 │
              └──────────────┬───────────────┘
                             │
                             ▼
              ┌──────────────────────────────┐
              │   第二层：状态检查            │
              │   !currentProjectId          │
              └──────────────┬───────────────┘
                             │
              ┌──────────────┴───────────────┐
              │   if (!currentProjectId)     │
              │       loadLatestProject();    │
              └──────────────┬───────────────┘
                             │
                             ▼
                        安全执行
```

**完整实现示例**：

```typescript
import { useEffect, useRef } from 'react';
import { useBuilderStore } from '@/store/useBuilderStore';

export const AppContent: React.FC = () => {
  // 从 store 获取状态值（不是方法引用）
  const currentProjectId = useBuilderStore((state) => state.currentProjectId);
  
  // 获取方法引用（仅用于调用，不用于依赖）
  const loadLatestProject = useBuilderStore((state) => state.loadLatestProject);

  // 第一层保护：useRef 标记
  const hasLoadedInitialProject = useRef(false);

  useEffect(() => {
    // 第一层检查：是否已执行过
    if (hasLoadedInitialProject.current) {
      return;
    }
    hasLoadedInitialProject.current = true;

    // 第二层检查：是否已有项目
    if (!currentProjectId) {
      const hasProject = loadLatestProject();
      if (hasProject) {
        toast.info('已恢复上次编辑的项目');
      }
    }
  }, []);  // ✅ 空依赖数组

  // ... 其他代码
};
```

**store 层的配合**：

```typescript
loadLatestProject: () => {
  // Store 层也应该有保护
  const { currentProjectId } = get();
  
  if (currentProjectId) {
    return false;  // 已有项目，跳过
  }
  
  // ... 加载逻辑
};
```

**为什么需要双重保护**：

| 场景 | 保护层 | 作用 |
|-----|-------|------|
| 组件首次挂载 | `useRef` | 确保只执行一次 |
| 组件重新挂载（如导航） | `currentProjectId` | 如果之前已经加载了项目，跳过 |
| Store 内部被错误调用 | Store 层检查 | 最后一道防线 |
