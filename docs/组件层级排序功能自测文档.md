# 组件层级排序功能自测文档

## 1. 功能概述

组件层级排序功能允许用户调整画布上组件的层叠顺序。层级由组件在数组中的索引位置决定：
- **索引 0**：最底层（z-index 最低）
- **最大索引**：最顶层（z-index 最高）

### 1.1 支持的操作

| 操作 | 功能描述 |
|------|---------|
| **上移** | 将组件索引 +1（视觉上移一层，更靠前显示） |
| **下移** | 将组件索引 -1（视觉上移一层，更靠后显示） |
| **置顶** | 将组件移到同级组件的最后一个索引（最顶层） |
| **置底** | 将组件移到同级组件的索引 0（最底层） |

### 1.2 显示位置

层级控制按钮出现在两个位置：
1. **Toolbar（工具栏）**：撤销/重做按钮右侧
2. **属性面板**：选中组件后，复制/删除按钮下方

### 场景 1：3 个组件同时存在时上移/下移的索引变化验证

#### 测试步骤

1. **准备测试环境**
   - 打开编辑器页面
   - 从左侧组件库拖拽 3 个组件到画布
   - 记录组件添加顺序：组件 A（第1个）、组件 B（第2个）、组件 C（第3个）

2. **初始状态验证**
   - 选中组件 A
   - 确认属性面板显示：`层级：1 / 3`
   - 选中组件 B
   - 确认属性面板显示：`层级：2 / 3`
   - 选中组件 C
   - 确认属性面板显示：`层级：3 / 3`

3. **上移操作测试**
   - 选中组件 A（层级 1）
   - 点击「上移」按钮
   - 确认层级显示变为：`层级：2 / 3`
   - 确认组件 A 现在显示在组件 B 上方

4. **下移操作测试**
   - 选中组件 C（层级 3）
   - 点击「下移」按钮
   - 确认层级显示变为：`层级：2 / 3`
   - 确认组件 C 现在显示在组件 B 下方

5. **最终顺序验证**
   - 查看组件层级关系：组件 A (2)、组件 C (2)、组件 B (1)
   - 实际显示顺序（从下到上）：组件 B → 组件 A/组件 C

#### 预期结果

| 操作 | 原始顺序 | 操作后顺序 |
|------|---------|-----------|
| 组件 A 上移 | [A(1), B(2), C(3)] | [B(1), A(2), C(3)] |
| 组件 C 下移 | [B(1), A(2), C(3)] | [B(1), C(2), A(3)] |

### 场景 5：Container 内子组件的层级调整

#### 测试步骤

1. **准备测试环境**
   - 打开编辑器页面
   - 从左侧组件库拖拽 1 个 Container 组件到画布
   - 在 Container 内拖拽 3 个组件（文本、按钮、图片）
   - 在 Canvas 根级别再拖拽 2 个组件（外部组件 X、Y）

2. **验证初始状态**
   - 组件结构：
     ```
     根级别: [Container, X, Y]
     Container 内: [文本, 按钮, 图片]
     ```
   - 选中 Container 内的「文本」组件
   - 确认层级显示：`层级：1 / 3`（Container 内的层级，不是全局）
   - 选中根级别的「X」组件
   - 确认层级显示：`层级：1 / 3`（根级别的层级）

3. **Container 内子组件上移测试**
   - 选中 Container 内的「文本」组件（层级 1）
   - 点击「上移」按钮
   - 确认层级显示变为：`层级：2 / 3`
   - 确认根级别的组件 X、Y 位置没有变化

4. **Container 内子组件置顶测试**
   - 选中 Container 内的「按钮」组件
   - 点击「置顶」按钮
   - 确认层级显示变为：`层级：3 / 3`
   - 确认按钮现在显示在 Container 内所有子组件的最上方
   - 确认根级别的组件 X、Y 层级显示不变

5. **验证隔离性**
   - Container 内的层级调整不会影响 Container 外部的组件
   - Container 外部的组件层级调整不会影响 Container 内部的子组件

#### 预期结果

| 操作范围 | 影响范围 | 不影响范围 |
|---------|---------|-----------|
| Container 内子组件上移 | Container 内 children 数组 | 根级别组件数组、其他 Container |
| Container 内子组件置顶 | Container 内 children 数组 | 根级别组件数组、其他 Container |
| 根级别组件上移 | 根级别 components 数组 | 所有 Container 的 children 数组 |

**层级计算规则**：
- `getComponentLayerInfo()` 返回的是**同级组件**中的层级
- 同级组件由 `parentId` 决定
- `parentId = null`：根级别组件
- `parentId = Container ID`：该 Container 内的子组件

### 场景 6：撤销/重做对层级变更的恢复

#### 测试步骤

1. **准备测试环境**
   - 打开编辑器页面
   - 从左侧组件库拖拽 3 个组件到画布
   - 记录初始顺序：A(1), B(2), C(3)

2. **记录初始状态**
   - 确认撤销按钮：禁用状态（canUndo = false）
   - 确认重做按钮：禁用状态（canRedo = false）

3. **执行层级操作**
   - 选中组件 A，点击「上移」（操作 1）
   - 选中组件 C，点击「置顶」（操作 2）
   - 选中组件 B，点击「下移」（操作 3）

4. **验证可撤销**
   - 确认撤销按钮：启用状态（canUndo = true）
   - 确认重做按钮：禁用状态（canRedo = false）

5. **执行撤销操作**
   - 点击「撤销」按钮（撤销操作 3）
   - 确认组件 B 回到之前的层级
   - 再次点击「撤销」按钮（撤销操作 2）
   - 确认组件 C 回到之前的层级
   - 再次点击「撤销」按钮（撤销操作 1）
   - 确认组件 A 回到初始层级（1 / 3）
   - 确认所有组件恢复到初始顺序

6. **验证可重做**
   - 确认重做按钮：启用状态（canRedo = true）
   - 确认撤销按钮：禁用状态（canUndo = false）

7. **执行重做操作**
   - 点击「重做」按钮（重做操作 1）
   - 再次点击「重做」按钮（重做操作 2）
   - 再次点击「重做」按钮（重做操作 3）
   - 确认所有组件恢复到操作后的顺序

#### 预期结果

| 阶段 | 操作 | canUndo | canRedo | 组件顺序 |
|------|------|---------|---------|---------|
| 初始 | 无 | false | false | A(1), B(2), C(3) |
| 操作后 | 3 次层级操作 | true | false | 最终顺序 |
| 撤销后 | 3 次撤销 | false | true | A(1), B(2), C(3) |
| 重做后 | 3 次重做 | true | false | 最终顺序 |

**历史记录机制**：
- 每次层级操作都会调用 `pushHistory()`
- 撤销时恢复 `currentIndex - 1` 的状态
- 重做时恢复 `currentIndex + 1` 的状态

   - 选中组件 B
   - 点击「删除」按钮
   - 选中组件 A：层级应该变为 1 / 2
   - 选中组件 C：层级应该变为 2 / 2（因为 B 被删除了）

5. **撤销删除测试**
   - 点击「撤销」按钮
   - 确认组件 B 重新出现
   - 选中组件 A：层级应该恢复为 1 / 3
   - 选中组件 B：层级应该恢复为 2 / 3
   - 选中组件 C：层级应该恢复为 3 / 3

#### 预期结果

| 操作 | 组件数量 | 组件 A 层级 | 组件 B 层级 | 组件 C 层级 |
|------|---------|------------|------------|------------|
| 初始 | 2 | 1 / 2 | 2 / 2 | - |
| 添加 C | 3 | 1 / 3 | 2 / 3 | 3 / 3 |
| 删除 B | 2 | 1 / 2 | - | 2 / 2 |
| 撤销删除 | 3 | 1 / 3 | 2 / 3 | 3 / 3 |

**层级计算逻辑**：
- `currentLayer = index + 1`（索引从 0 开始，层级从 1 开始显示）
- `totalLayers = 同级组件总数`
- 每次组件增删或层级调整后，`getComponentLayerInfo()` 都会重新计算

## 5. 技术实现要点

### 5.1 Store 层核心方法

```typescript
// 层级操作
moveUp(id: string): void      // 索引 +1
moveDown(id: string): void    // 索引 -1
moveToTop(id: string): void   // 移到最后一个索引
moveToBottom(id: string): void // 移到索引 0

// 状态查询
canMoveUp(id: string): boolean    // 是否可以上移
canMoveDown(id: string): boolean  // 是否可以下移
getComponentLayerInfo(id: string): {
  currentLayer: number;   // 当前层级（index + 1）
  totalLayers: number;    // 同级总层数
} | null
```

### 5.2 层级与索引关系

```
视觉显示（从上到下）：
  组件 C (index 2, layer 3) ← 最顶层，置顶时移到这里
  组件 B (index 1, layer 2)
  组件 A (index 0, layer 1) ← 最底层，置底时移到这里

数组存储（components 数组）：
  [组件 A, 组件 B, 组件 C]
    ↑        ↑        ↑
  index 0  index 1  index 2
  layer 1  layer 2  layer 3
```

### 5.3 Container 子组件隔离

```typescript
// 通过 parentId 区分同级组件
const getSiblingsList = (
  components: ComponentSchema[],
  parentId: string | null
): ComponentSchema[] => {
  if (parentId === null) {
    return components;  // 根级别组件
  }
  // Container 内子组件
  const parent = findComponentById(components, parentId);
  if (parent && isContainerComponent(parent) && parent.children) {
    return parent.children;
  }
  return [];
};
```

## 6. 注意事项

1. **层级显示规则**：
   - 层级数字从 1 开始（不是从 0）
   - `currentLayer = index + 1`
   - `totalLayers = 同级组件数量`

2. **边界条件**：
   - 单个组件时，所有移动操作都不生效
   - 最顶层时，上移/置顶不生效
   - 最底层时，下移/置底不生效

3. **历史记录**：
   - 每次层级操作都会记录到历史
   - 支持撤销/重做
   - 与其他操作（添加、删除、更新）共享历史栈

4. **Container 隔离**：
   - Container 内的层级调整不会影响外部组件
   - 外部组件的层级调整不会影响 Container 内部
   - 每个 Container 维护独立的子组件层级
