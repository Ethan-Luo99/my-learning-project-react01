# 画布交互增强功能自测文档

## 概述

本文档记录了画布交互增强功能（缩放、多选、对齐线）的测试场景和预期结果。

---

## 一、缩放功能 (Resize)

### 1.1 基本缩放功能测试

#### 测试场景 1.1.1: 选中组件后 8 个手柄正确显示

**前置条件**：
- 画布上至少有一个组件
- 组件未被选中

**操作步骤**：
1. 点击组件选中该组件

**预期结果**：
- [ ] 组件四周显示蓝色选中框
- [ ] 组件四周和四角显示 8 个 resize 手柄
- [ ] 8 个手柄分别位于：上、下、左、右、左上、右上、左下、右下
- [ ] 手柄为可拖拽的小方块或圆点

---

#### 测试场景 1.1.2: 拖拽右下角手柄调整宽度和高度

**前置条件**：
- 画布上有一个已选中的组件
- 组件显示 8 个 resize 手柄

**操作步骤**：
1. 将鼠标悬停在右下角手柄上
2. 按住鼠标左键并向右下方拖拽
3. 释放鼠标左键

**预期结果**：
- [ ] 拖拽过程中组件实时调整大小
- [ ] 组件宽度随拖拽方向增加
- [ ] 组件高度随拖拽方向增加
- [ ] 组件保持网格对齐（16px 网格）
- [ ] 释放鼠标后组件保持新的尺寸

---

#### 测试场景 1.1.3: 拖拽左上角手柄调整宽度和高度

**前置条件**：
- 画布上有一个已选中的组件
- 组件初始位置：x=100, y=100
- 组件初始尺寸：width=200, height=100

**操作步骤**：
1. 将鼠标悬停在左上角手柄上
2. 按住鼠标左键并向左上方拖拽
3. 释放鼠标左键

**预期结果**：
- [ ] 拖拽过程中组件实时调整大小
- [ ] 组件宽度减少
- [ ] 组件高度减少
- [ ] 组件位置 (x, y) 向右下方移动（保持右下角固定）
- [ ] 组件保持网格对齐

---

#### 测试场景 1.1.4: 拖拽左右手柄只调整宽度

**前置条件**：
- 画布上有一个已选中的组件

**操作步骤**：
1. 将鼠标悬停在右边手柄上
2. 按住鼠标左键并向右拖拽
3. 释放鼠标左键

**预期结果**：
- [ ] 组件宽度增加
- [ ] 组件高度保持不变
- [ ] 组件高度保持不变

---

#### 测试场景 1.1.5: 拖拽上下手柄只调整高度

**前置条件**：
- 画布上有一个已选中的组件

**操作步骤**：
1. 将鼠标悬停在下方手柄上

### 1.2 最小尺寸限制测试

#### 测试场景 1.2.1: 缩放到最小尺寸时停止

**前置条件**：
- 画布上有一个已选中的组件
- 组件初始尺寸：width=200, height=100

**操作步骤**：
1. 将鼠标悬停在右下角手柄上
2. 按住鼠标左键并向左上方拖拽
3. 持续拖拽直到组件无法再缩小
4. 释放鼠标左键

**预期结果**：
- [ ] 组件宽度最小到一定尺寸时停止缩小
- [ ] 最小宽度为 16px
- [ ] 最小高度为 16px
- [ ] 拖拽到最小尺寸后，继续拖拽不会进一步缩小

---

#### 测试场景 1.2.2: 容器组件缩放时子组件不受影响

**前置条件**：
- 画布上有一个 Container 组件
- Container 内部有一个子组件（如 Text 或 Button）
- Container 已被选中

**操作步骤**：
1. 将鼠标悬停在 Container 的右下角手柄上
2. 按住鼠标左键并向右下方拖拽
3. 释放鼠标左键

**预期结果**：
- [ ] Container 的宽度和高度增加
- [ ] 内部子组件的尺寸保持不变
- [ ] 内部子组件的位置保持不变
- [ ] 内部子组件仍然在 Container 内

---

### 1.3 撤销/重做测试

#### 测试场景 1.3.1: 撤销操作恢复到缩放前尺寸

**前置条件**：
- 画布上有一个组件
- 组件初始尺寸：width=200, height=100

**操作步骤**：
1. 选中组件

3. 按下撤销快捷键（Ctrl+Z 或 Cmd+Z）

**预期结果**：
- [ ] 组件尺寸恢复到缩放前的尺寸
- [ ] 组件宽度恢复到 200
- [ ] 组件高度恢复到 100
- [ ] 可以重做（Ctrl+Y 或 Cmd+Shift+Z）

---

#### 测试场景 1.3.2: 重做操作恢复到缩放后尺寸

**前置条件**：
- 已完成场景 1.3.1 的操作

**操作步骤**：
1. 按下重做快捷键（Ctrl+Y 或 Cmd+Shift+Z）

**预期结果**：
- [ ] 组件尺寸恢复到缩放后的尺寸
- [ ] 组件宽度恢复到 300
- [ ] 组件高度恢复到 200

---

### 1.4 网格对齐测试

## 三、对齐线功能 (AlignmentGuides)

### 3.1 基本对齐检测测试

#### 测试场景 3.1.1: 拖拽时显示对齐辅助线

**前置条件**：
- 画布上有至少两个组件
- 组件 A: x=100, y=100, width=100, height=100
- 组件 B: x=300, y=300, width=100, height=100
- 组件 B 被选中

**操作步骤**：
1. 拖拽组件 B 向左移动
2. 当组件 B 的左边缘接近组件 A 的左边缘（x=100）时，观察
3. 继续拖拽直到 x 接近 100

**预期结果**：
- [ ] 当组件 B 的 x 接近 100 时（在 8 像素范围内），显示蓝色垂直对齐线
- [ ] 对齐线为细蓝色线条
- [ ] 对齐线穿过组件 A 的左边缘位置（x=100）
- [ ] 组件 B 自动吸附到 x=100

---

### 5.2 注意事项

1. **网格对齐 vs 对齐线对齐
   - 对齐线对齐优先于网格对齐
   - 如果检测到对齐线时，组件会吸附到对齐线位置
   - 如果没有检测到对齐线，组件会保持网格对齐

2. **最小尺寸限制**
   - 组件最小尺寸为 16x16 像素
   - 缩放时不会小于这个尺寸

3. **多选拖拽对齐
   - 多选拖拽时，以第一个选中组件为基准检测对齐
   - 所有选中组件作为整体移动

4. **撤销/重做**
   - 缩放、移动、删除操作都支持撤销/重做
   - 多选状态也会随撤销/重做恢复

5. **对齐容差**
   - 对齐容差为 8 像素
   - 在容差范围内会自动吸附
