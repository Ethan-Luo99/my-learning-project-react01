# 模块 1.6 补充：ComponentRenderer 优化与交互完善

## 背景说明

当前 ComponentRenderer 已实现基础渲染，但存在多个需要优化的问题：
1. 类型不匹配：ComponentRenderer 使用了不存在的 children 属性
2. 属性传递混乱：props 展开与具体属性混用
3. 缺少事件拦截：组件内部点击会冒泡到外层
4. 缺少空状态处理：未渲染组件时应有友好提示
5. 缺少组件标签显示：编辑模式下应显示组件类型标签

---

## 题目 1.6.8：修复 ComponentRenderer 类型与渲染问题

### 题目描述

修复 ComponentRenderer 中的类型错误和渲染逻辑问题。

### 具体要求

#### 1. 修复类型定义不匹配问题

**问题**：
- ComponentRenderer 使用了 `children` 属性，但 ComponentSchema 接口中只有 Container 类型才有 children
- Text 组件使用了 `props.children`，但 TextProps 中可能没有这个属性

**解决方案**：
- Text 组件应使用 `props.content` 或 `props.children` 作为文本内容
- 只有 Container 类型才需要渲染 children
- Button/Image 不应该有 children

#### 2. 修复属性传递混乱问题

**问题**：
```typescript
// 当前代码存在问题
<Button style={styles} {...props}>  // style 和 props 可能冲突
  {props.children || '按钮'}
</Button>
```

**解决方案**：
- 不要同时使用 `style={styles}` 和 `{...props}`
- 应该将 styles 合并到 className 或使用 style 属性
- 明确提取需要的 props，避免过度展开

#### 3. 修复事件冒泡问题

**问题**：
- ComponentRenderer 的 onClick 只包裹在 div 上
- 如果组件内部有按钮等可点击元素，会触发内部事件而非选中事件

**解决方案**：
- 在编辑模式下，阻止组件内部所有点击事件冒泡
- 或者使用 `pointer-events` 控制

#### 4. 添加组件标签显示

**需求**：
- 在编辑模式下，每个组件上方显示类型标签（如 "按钮"、"文本"）
- 选中时标签高亮
- 预览模式下隐藏标签

### 验收标准

- ✅ Text 组件正确显示文本内容
- ✅ Button 组件正确显示按钮文字
- ✅ Container 组件正确渲染子组件
- ✅ 点击组件能正确触发选中
- ✅ 组件标签在编辑模式显示
- ✅ 无 TypeScript 类型错误

## 题目 1.6.10：完善 Store 的 Mock 数据初始化

### 题目描述

当前 Store 使用了 `MOCK_CANVAS_DATA` 作为初始数据，但需要优化。

### 具体要求

#### 1. 修复 Mock 数据类型

**问题**：
- MOCK_CANVAS_DATA 中的组件类型可能不匹配
- props 结构可能不符合 ComponentSchema 定义

**解决方案**：
- 确保 Mock 数据严格符合 ComponentSchema 类型
- 每个组件的 props 应该使用该组件类型的默认属性

#### 2. 添加空画布初始化选项

**需求**：
- 提供两种方式初始化：
  1. 使用 Mock 数据（用于演示）
  2. 空画布（用于实际使用）
- 可以通过环境变量或参数控制

#### 3. 添加数据验证

在 setComponents 时：
- 验证组件数据结构
- 过滤掉无效的组件
- 记录警告日志

### 验收标准

- ✅ Mock 数据符合类型定义
- ✅ 可以切换空画布/Mock 数据
- ✅ 数据验证工作正常

## 题目 1.6.11：实现组件悬停预览效果

### 题目描述

在编辑模式下，鼠标悬停在组件上时显示预览效果。

### 具体要求

#### 1. 添加悬停状态

在 Canvas 中：
- 跟踪当前悬停的组件 ID
- 传递给 ComponentRenderer

#### 2. 悬停视觉反馈

ComponentRenderer 中：
- 悬停时显示虚线边框
- 背景色轻微变化
- 显示组件类型标签

#### 3. 区分悬停与选中

- 悬停：虚线边框 + 浅色背景
- 选中：实线边框（主题色）+ 较深背景
- 同时悬停和选中：以选中状态为准

### 验收标准

- ✅ 鼠标悬停组件有视觉反馈
- ✅ 悬停与选中状态区分明显
- ✅ 状态切换流畅无闪烁
