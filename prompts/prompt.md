# 低代码/可视化搭建平台 - 出题方向规划

## 项目概述
**技术栈**: React + TypeScript + DnD Kit + Zustand + TailwindCSS  
**业务场景**: H5 营销活动页面可视化搭建器  
**训练目标**: 考察复杂组件树管理、拖拽交互逻辑、JSON 协议设计能力

---

## 出题模块划分

### 模块一：项目初始化与基础架构搭建
**目标**: 从零开始搭建 React 项目，完成基础配置和依赖安装

1. **1.1 项目脚手架搭建**
   - 使用 Vite + React + TypeScript 初始化项目
   - 配置项目目录结构
   - 配置基础 ESLint + Prettier

2. **1.2 依赖安装与配置**
   - 安装并配置 TailwindCSS
   - 安装 Zustand 状态管理库
   - 安装 DnD Kit 拖拽库
   - 安装其他工具库（uuid、lodash 等）

3. **1.3 基础组件库搭建**
   - 创建基础 UI 组件目录结构
   - 实现 Button 基础组件
   - 实现 Text 文本组件
   - 实现 Image 图片组件

4. **1.4 路由与页面布局**
   - 配置主页面布局（左侧组件库、中间画布、右侧属性面板）
   - 实现响应式布局框架
   - 配置基础样式变量

---

### 模块二：组件协议设计与数据模型
**目标**: 定义组件 JSON Schema 协议，建立数据流转基础

5. **2.1 ComponentSchema 接口定义**
   - 定义组件基础接口（id、type、props、styles）
   - 定义嵌套组件结构（children）
   - 定义事件配置接口

6. **2.2 组件类型枚举与映射**
   - 定义组件类型枚举（Button、Text、Image、Container 等）
   - 建立组件类型到 React 组件的映射表
   - 实现组件注册机制

7. **2.3 Mock 数据设计**
   - 设计基础页面模板数据
   - 设计嵌套组件示例数据
   - 设计属性配置示例数据

8. **2.4 数据验证工具函数**
   - 实现 Schema 数据校验函数
   - 实现组件嵌套规则验证
   - 实现数据类型转换工具

---

### 模块三：画布渲染器实现
**目标**: 实现 JSON 到 React 组件的递归渲染引擎

9. **3.1 基础渲染器 Renderer**
   - 实现接收 ComponentSchema[] 的渲染组件
   - 实现单层组件渲染逻辑
   - 正确应用 styles 样式

10. **3.2 递归渲染支持**
    - 实现 children 递归渲染
    - 处理空 children 边界情况
    - 添加 React key 属性优化

11. **3.3 样式应用引擎**
    - 实现内联样式转换
    - 支持 TailwindCSS 类名应用
    - 处理样式优先级

12. **3.4 渲染性能优化**
    - 实现 React.memo 优化
    - 避免不必要的重渲染
    - 虚拟列表支持（长列表场景）

---

### 模块四：拖拽交互系统
**目标**: 实现组件拖拽、排序、嵌套等核心交互

13. **4.1 基础拖拽功能**
    - 实现 DraggableItem 组件
    - 处理鼠标按下、移动、释放事件
    - 实现拖拽时的视觉反馈

14. **4.2 落点计算与插入逻辑**
    - 计算拖拽目标位置坐标
    - 实现组件插入位置判断
    - 处理边界情况（头部、尾部、中间）

15. **4.3 嵌套拖拽支持**
    - 实现容器组件接收子组件
    - 拖拽时的嵌套高亮提示
    - 限制非法嵌套（如按钮内不能嵌套容器）

16. **4.4 拖拽状态同步**
    - 拖拽过程中实时更新 Zustand store
    - 处理拖拽取消/回滚
    - 拖拽性能优化（节流处理）

---

### 模块五：状态管理系统

19. **5.3 撤销/重做功能**

    - 优化内存占用（限制历史步数）

20. **5.4 数据持久化**

## 出题策略说明

### 难度梯度设计
- **模块一~三**: 基础入门（适合初级水平）
- **模块四~六**: 核心功能（适合中级水平）
- **模块七~八**: 增强功能（适合中高级水平）
- **模块九~十**: 完整交付（适合高级水平）

### 每题拆解原则
每个小模块可继续拆分为 2-4 个具体 prompt：
1. **基础实现**: 完成核心功能
2. **边界处理**: 处理异常/边界情况
3. **优化改进**: 性能/可维护性优化
4. **测试验证**: 编写测试用例

### 预期题目总数
- **基础版本**: 40 个模块 × 2 题 = **80 题**
- **完整版本**: 40 个模块 × 3 题 = **120 题**
- **进阶版本**: 40 个模块 × 4 题 = **160 题**
